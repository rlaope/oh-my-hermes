# CodeGraph for this repository

OMH is initialized for the external CodeGraph project (`@colbymchenry/codegraph`).
CodeGraph builds a local, queryable code-intelligence index for this repository so
agents can inspect symbols, file structure, callers/callees, and impact radius
without reading the whole tree first.

CodeGraph is a local navigation aid only. Its output is prepared code context; it
is not execution, review, CI, merge-readiness, or merge evidence.

## What is committed

Only the ignore guard is committed:

```text
.codegraph/.gitignore
```

The actual index is machine-local and must not be committed:

```text
.codegraph/codegraph.db
```

That database is generated by `codegraph init` and refreshed by `codegraph sync`.

## Prerequisite

Use Node/npm through `npx`; no Python package dependency is added to OMH.

Check that the CLI is available:

```sh
npx @colbymchenry/codegraph --help
```

## Build the local index

From the repository root:

```sh
npx @colbymchenry/codegraph init .
```

This creates `.codegraph/codegraph.db` locally. On the current repository this was
observed to index Python and YAML files and report repository-level symbol and
edge statistics.

Check status:

```sh
npx @colbymchenry/codegraph status .
```

Refresh after code changes:

```sh
npx @colbymchenry/codegraph sync .
```

If the index is broken or stale beyond repair, rebuild it:

```sh
npx @colbymchenry/codegraph index .
```

Remove the local index if needed:

```sh
npx @colbymchenry/codegraph uninit .
```

## Useful queries

Search for symbols or files:

```sh
npx @colbymchenry/codegraph query codegraph --path .
npx @colbymchenry/codegraph query runtime_records --path .
```

Explore an area before opening lots of files:

```sh
npx @colbymchenry/codegraph explore routing --path .
npx @colbymchenry/codegraph explore "goal lifecycle" --path .
```

Inspect one symbol or file:

```sh
npx @colbymchenry/codegraph node cmd_codegraph_build --path .
npx @colbymchenry/codegraph node src/commands/main.py --path .
```

Find callers/callees or impact radius:

```sh
npx @colbymchenry/codegraph callers validate_run_record --path .
npx @colbymchenry/codegraph callees route_chat_message --path .
npx @colbymchenry/codegraph impact create_run --path .
```

List indexed project files:

```sh
npx @colbymchenry/codegraph files --path .
```

## Agent usage

Before broad code exploration, agents should prefer CodeGraph for a first pass:

1. `status` to confirm the index exists and is fresh.
2. `query` or `explore` to identify likely files/symbols.
3. Read only the files needed for the task.
4. Run normal tests/verification before making claims.

CodeGraph can reduce exploration cost, but it does not replace source reads,
tests, review, or CI.

## Hermes MCP note

CodeGraph can print a Hermes MCP config snippet:

```sh
npx @colbymchenry/codegraph install --print-config hermes
```

At the time this repository was initialized, CodeGraph's installer reported that
Hermes Agent does not support project-local installation for this target. Do not
mutate a developer's global `~/.hermes/config.yaml` from this repository. If a
developer wants global Hermes MCP integration, they should apply the printed
snippet intentionally in their own environment and restart Hermes.
