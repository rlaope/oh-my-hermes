Flagship lane

Image gen that stays source-aware.

visual-summary helps Hermes prepare image-generation prompt cards for meeting notes, reports, GitHub PRs, issues, research briefings, and release updates. OMH shapes the card before a connected image tool runs, then records image generation, visual QA, attachment, and delivery only when those events are observed.

Formats follow the source

The visual system is not one fixed grid. OMH selects a visual format and aspect ratio that match the source, then keeps the shared proof structure stable: summary, key points, why it matters, next action, caveats, and observed evidence.

Long-scroll digest cards for metrics, observations, caveats, and actions.

What is new, who should care, upgrade notes, and confidence level.

End-to-end flow

Hermes can use this lane when a chat summary needs to become a shareable visual. The wrapper can show the prompt card, copy the prompt into an already connected image tool, then record what happened back into OMH.

source visual_prompt_card/v1 connected image tool visual_observation/v1

Evidence ladder

  • Prompt prepared means copy, layout, language, negative prompt, and QA checklist are ready.
  • Image generated means a connected image tool produced a concrete image.
  • Visual QA passed means somebody checked readability, fit, source alignment, and safety.
  • Attached or delivered means a wrapper or operator shared the image.
  • Recorded in OMH means a matching visual_observation/v1 event exists.

Try the backend contract

Normal users should ask Hermes for the card in chat. Operators and wrappers can call the deterministic backend to preview the exact contract shape without contacting an image provider.

omh visual prompt-card --kind github_pr --section summary:What_changed:Setup_copy_now_matches_the_site omh visual prompt-card --kind report --section summary:Executive_summary:Weekly_metrics_changed omh visual observe --card-id <id> --type generated_image_observed --path ./card.png --summary "Generated with connected image tool"

The first two commands prepare visual_prompt_card/v1. They are not generated image evidence. The observation command records one explicit event after the external image step actually happens.
