Oh My Hermes

A stronger operating layer for the Hermes you already use.

Capability system

Start with the job, not the command.

OMH groups a large skill catalog into six human-readable families. Hermes can recommend the smallest useful lane while keeping every trigger, quality bar, provider requirement, and claim boundary inspectable.

01Plan and decide 02Learn and gather 03Create materials and visuals 04Delegate coding and ship 05Operate and observe 06Retain knowledge

Route with context

Normal language in. A bounded first move out.

English, Korean, Japanese, Chinese, Spanish, French, German, and Hindi operator requests can be classified locally. OMH returns a family, skill, owner, next action, and what still is not evidence.

  • Deep interview, planning, durable goals, and research
  • Media, data, web, paper, and best-practice routes
  • Guarded intent matching to reduce common-name overroutes

Create with quality gates

Files, visuals, reports, and interfaces get the right production lane.

OMH distinguishes source analysis from generation, and a prepared visual or document brief from an observed artifact. Frontend, accessibility, PDF, slide, report, and image requests carry format-specific review expectations.

  • Frontend publishing plus visual and accessibility QA
  • Image, report, deck, document, and deliverable packages
  • External image, video, browser, and connector providers stay explicit

Coding handoff

Use the coding environment you trust, with more context.

Codex, Claude Code, Hermes runtime, and generic executors are first-class owners. OMH prepares scope, non-goals, acceptance criteria, worktree guidance, available local skills, review expectations, and evidence gates before dispatch is claimed.

  • Executor readiness and skill-aware prompt preparation
  • Dynamic typed workflows across models, tools, wrappers, and agents
  • Builder, verifier, reviewer, docs, and PR responsibility lanes

Operate, learn, remember

Turn work into inspectable operating context.

OMH can prepare service-quality analysis from an external metric provider, review project-local memory, connect a wiki provider, inspect sessions, and turn weak routes into learning candidates. Provider access and outcome improvement remain separate observed claims.

  • Service quality, release, reliability, and incident review
  • Reviewed project memory and external knowledge-provider boundaries
  • Workflow learning, missed-route regression, and session recovery

Capability impact

Measure each claim at its own boundary.

Install

Three commands for people. The rest belongs to the agent layer.

Set up OMH, keep it current, and check its health. After that, ask Hermes for the work in normal language.

# Install OMH once curl -fsSL https://raw.githubusercontent.com/rlaope/oh-my-hermes/main/install.sh | sh # Human OMH commands omh setup omh update omh doctor # Optional Hermes-native skill tap hermes skills tap add rlaope/oh-my-hermes hermes skills install rlaope/oh-my-hermes/skills/oh-my-hermes --yes
